# 📊 Data Mining — Unit 5 Simple Notes

---

## 1. Classification & Prediction kya hota hai?

Data mining mein hum data se **patterns dhundhte hain** aur future ke baare mein **predictions** karte hain.

**Classification** ka matlab hai — kisi cheez ko ek **category mein dalna.** Jaise ek email ko "spam" ya "not spam" mein dalna, ya ek patient ko "bimaar" ya "theek" mein categorize karna.

**Prediction** ka matlab hai — koi **numerical value estimate karna.** Jaise ghar ki price predict karna ya kal ka temperature batana.

---

## 2. Issues Related to Classification & Prediction

Classification aur prediction karte waqt kaafi saari problems aati hain:

**Missing Data** — Kaafi baar dataset mein kuch values missing hoti hain. Jaise kisi patient ki age record nahi hai. Aisa data handle karna mushkil hota hai.

**Noisy Data** — Data mein galat ya irrelevant information hoti hai jo model ko confuse karti hai. Jaise ek spam email jo bilkul normal lagti hai.

**Overfitting** — Model training data pe itna zyada fit ho jaata hai ki naye data pe galat predictions deta hai. Jaise ek student jo sirf purane papers ratta maar le par naye questions solve na kar paaye.

**Imbalanced Data** — Jab ek class ka data bahut zyada ho aur doosri ka bahut kam. Jaise 1000 normal emails aur sirf 10 spam emails — model normal ki taraf bias ho jaata hai.

**High Dimensionality** — Jab features (columns) bahut zyada hon toh model complex ho jaata hai aur slow bhi.

---

## 3. Attribute Selection Measures

Jab hum classification karte hain toh sabse pehla sawaal hota hai — **kaun sa attribute (feature) sabse important hai?** Isi ko decide karne ke liye Attribute Selection Measures use hote hain.

**Information Gain** — Yeh measure karta hai ki koi attribute data ko kitna **saaf tarike se divide** kar sakta hai. Jis attribute se sabse zyada "clarity" mile woh select hota hai. Decision Trees mein yahi use hota hai.

**Gini Index** — Yeh measure karta hai ki ek randomly chosen element kitni baar **galat classify** hoga. Jitna kam Gini Index utna better attribute.

**Gain Ratio** — Information Gain ka improved version hai. Kyunki Information Gain kabhi kabhi un attributes ko favor karta hai jinke bahut saare values hain — Gain Ratio is bias ko theek karta hai.

Sab ka ek hi goal hai — **best splitting attribute dhundhna** taaki classification accurate ho.

---

## 4. Decision Tree Induction

Decision Tree ek **tree jaisi structure** hoti hai jisme har node ek question hota hai aur har branch ek answer. Antmein ek leaf node hoti hai jo final class batati hai.

Jaise ek doctor sochta hai — "Kya patient ko bukhaar hai? Haan → Kya khansi bhi hai? Haan → Flu ho sakta hai."

**Kaise banta hai:** Sabse pehle woh attribute choose hota hai jiska **Information Gain ya Gini Index sabse best** ho. Usi pe pehla split hota hai. Phir har branch pe dobara yahi process hoti hai jab tak saare data points ek hi class mein na aa jayein ya koi attribute na bache.

**Fayde** — Samajhna aur visualize karna aasaan hai. Non-technical log bhi samajh sakte hain.

**Nuksaan** — Zyada complex trees mein **overfitting** ho sakti hai — isliye pruning ki jaati hai yaani extra branches kaat di jaati hain.

---

## 5. Bayesian Classification

Bayesian Classification **probability** ke basis pe kaam karta hai. Yeh **Bayes Theorem** use karta hai jo kehta hai — pichle experience ke basis pe kisi cheez ke hone ki kitni probability hai.

**Naive Bayes** sabse common Bayesian classifier hai. Isme assume kiya jaata hai ki saare features **ek doosre se independent** hain — isliye "Naive" kehte hain.

**Simple example:** Ek email spam hai ya nahi yeh decide karna. Agar email mein "free", "win", "prize" jaisi words hain toh pehle ke experience se pata hai ki aisi emails zyada tar spam hoti hain. Naive Bayes inhi probabilities ko multiply karke final decision leta hai.

**Fayde** — Fast hota hai, kam data mein bhi achha kaam karta hai, text classification ke liye bahut popular hai.

---

## 6. K-Nearest Neighbor (KNN) Classifier

KNN ek bahut **simple aur intuitive** classification method hai. Iska idea yeh hai — **"Jo jaise logon ke saath rehta hai, woh waisa hi hota hai."**

Kaam karne ka tarika — jab koi naya data point aata hai toh KNN uske **K sabse paas wale neighbors** dhundta hai jo pehle se classified hain. Phir majority vote hoti hai — jis class ke zyada neighbors hain naya point usi class mein jaata hai.

Jaise agar tumhara naya point 5 neighbors ke paas hai jisme 3 "spam" hain aur 2 "not spam" — toh naya point "spam" classify hoga.

**K ka value important hai** — agar K bahut chota ho toh noise ka effect zyada hoga, agar bahut bada ho toh boundaries blur ho jaayengi.

**Nuksaan** — Bade datasets mein slow hota hai kyunki har naye point ke liye saare data se distance calculate karni padti hai.

## 7. Overview of Advanced Features of Data Mining

Basic classification aur clustering ke alawa data mining mein kaafi **advanced capabilities** bhi hoti hain. Jab data simple tabular format mein nahi hota — jaise images, web pages, geographical data — tab special techniques ki zaroorat padti hai. Iske baad wale topics inhi advanced features ke baare mein hain.

---

## 8. Mining Complex Data Objects

Normal data mining simple rows aur columns wale data pe kaam karti hai. Lekin real world mein data **complex forms** mein hota hai jaise graphs, sequences, trees ya spatial objects.

**Mining complex data objects** ka matlab hai — in complex structures se bhi **meaningful patterns** nikalna. Jaise social network mein communities dhundhna, DNA sequences mein patterns nikalna, ya molecular structures analyze karna.

Yahan normal algorithms directly apply nahi hoti — special methods banaye jaate hain jo in structures ki **shape aur relationship** ko samjhein.

---

## 9. Spatial Databases

Spatial databases mein **geographical ya location-based data** store hota hai. Jaise maps, GPS coordinates, satellite images.

**Spatial data mining** is data se patterns nikalti hai. Jaise — "Kaunse areas mein crime rate zyada hai?", "Kahan naya hospital banana chahiye?", "Is sheher mein traffic kahan zyada hota hai?"

Yahan normal distance aur relationships kaam nahi karti — **geographical proximity** (paas hona) aur **spatial relationships** (kaunsa area kaunse ke andar hai) important hoti hain.

---

## 10. Multimedia Databases

Multimedia databases mein **text ke alawa** doosre types ka data hota hai — images, audio, video, animations.

Inse mining karna mushkil hota hai kyunki yeh data **unstructured** hota hai — directly compare ya search nahi kar sakte jaise numbers karte hain.

Isliye pehle in files se **features extract** ki jaati hain. Jaise image se color, texture aur shape nikalo, phir un features pe mining karo. Isi wajah se Google Images mein kisi bhi photo se milti julti photos dhundh sakte hain.

---

## 11. Mining World Wide Web

Web bahut badi aur **unstructured information** ka source hai. Web mining teen tarike se hoti hai:

**Web Content Mining** — Web pages ke actual content se information nikalना. Jaise news articles se topics classify karna ya product reviews se sentiment analyze karna.

**Web Structure Mining** — Web pages ke **links aur structure** se patterns nikalna. Google ka PageRank algorithm isi pe based hai — jis page pe zyada links point karti hain woh zyada important maana jaata hai.

**Web Usage Mining** — Users ke **browsing behavior** se patterns nikalna. Jaise kaunse pages zyada visit hote hain, users kahan click karte hain — isi se websites recommend karte hain aur ads target karte hain.
